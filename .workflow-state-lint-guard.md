# Workflow State Literals Guard — P451

This document defines the lint guard against workflow state literals.

## Rule: No Bare Workflow State Literals

**Scope**: All TypeScript files in `src/**/*.ts`

**Exemptions**:
- `src/core/workflow/state-names.ts` — the single source of truth module
- `src/core/workflow/smdl-loader.ts` — SMDL workflow definitions (schema)
- `tests/**/*` — test fixtures
- `scripts/` — operational/development helper scripts (lower enforcement)

**Forbidden Patterns**:
```typescript
// ❌ RFC workflow states
'DRAFT', "DRAFT"
'REVIEW', "REVIEW"
'DEVELOP', "DEVELOP"
'MERGE', "MERGE"
'COMPLETE', "COMPLETE"
'REJECTED', "REJECTED"
'DISCARDED', "DISCARDED"

// ❌ Hotfix workflow states
'TRIAGE', "TRIAGE"
'FIX', "FIX"
'DEPLOYED', "DEPLOYED"
'ESCALATE', "ESCALATE"
'WONT_FIX', "WONT_FIX"
'NON_ISSUE', "NON_ISSUE"

// ❌ Maturity levels
'new', "new"
'active', "active"
'mature', "mature"
'obsolete', "obsolete"
```

**Approved Replacement Patterns**:
```typescript
// ✅ Use RfcStates accessor
import { RfcStates } from '../core/workflow/state-names.ts';
if (stage === RfcStates.DRAFT) { ... }
if (stage === RfcStates.COMPLETE) { ... }

// ✅ Use Maturity constants
import { Maturity } from '../core/workflow/state-names.ts';
if (maturity === Maturity.MATURE) { ... }
const validMaturityValues = [Maturity.NEW, Maturity.ACTIVE, Maturity.MATURE, Maturity.OBSOLETE];

// ✅ Use predicates from state-names registry
import { isTerminal, isGateable, nextOnMature } from '../core/workflow/state-names.ts';
if (isTerminal('rfc', stage)) { ... }
```

## Enforcement Strategy

Since Biome (the current linter) does not support `no-restricted-literals` rules, enforcement is currently **manual/review-based**:

1. **Code Review**: Reviewers check for state literal usage in PRs
2. **Automated Check** (future): Consider migrating to ESLint with `@babel/eslint-parser` to enable:
   ```json
   {
     "rules": {
       "no-restricted-syntax": [
         "error",
         {
           "selector": "Literal[value=/^(DRAFT|REVIEW|DEVELOP|MERGE|COMPLETE|TRIAGE|DEPLOYED|FIX|WONT_FIX|NON_ISSUE|ESCALATE|REJECTED|DISCARDED)$/]",
           "message": "Use RfcStates or state-names predicates instead of bare state literals"
         },
         {
           "selector": "Literal[value=/^(new|active|mature|obsolete)$/]",
           "message": "Use Maturity constants instead of bare maturity literals"
         }
       ]
     }
   }
   ```

## Exceptions

**Display strings and text output** (Category B) may use state name strings for user-facing output:
- Error messages that print state names for humans
- UI display arrays that list available statuses
- Log messages that show current state in English

These are **not state machine logic** and do not need constants, but should still prefer constants where feasible.

## Validation Checklist

After migration, verify:
```bash
# Should return ZERO matches outside state-names.ts and SMDL
rg -n "['\"](DRAFT|REVIEW|DEVELOP|MERGE|COMPLETE|TRIAGE|DEPLOYED|FIX|WONT_FIX|NON_ISSUE|ESCALATE|REJECTED|DISCARDED)['\"]" src/ --type ts

# Should return ZERO matches outside definition modules
rg -n "['\"](mature|active|obsolete|new)['\"]\\s*[)=,]" src/ --type ts
```
